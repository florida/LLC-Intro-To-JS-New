Syntax Highlighting

Code highlighting is provided by highlight.js and will be applied to any code wrapped in <pre><code></code></pre>. The syntax highlighting theme files can be found in framework > css > highlightjs-themes. If you wish to change it, add the css file there and update the css link. Visit highlight.js to see the full list of supported languages, themes and other usage information.

JavaScript

var greeting = "Hello!";

function sayHello() {
    console.log(greeting);
}

sayHello();

##CSS Customizations The CSS for this template is written in [Sass](http://sass-lang.com/). For your forked copy of these slides, feel free to use either Sass or the compiled CSS files. Just note that if you pull in any styles updates from the repo, the CSS files will be recompiled. If you want to change the intro slide background image (or not use an image at all), change the style using the `.intro` class. The text in the slide can automatically scale based on window size. If you want to turn this feature "on" or off", in **framework > sass > _variables.scss** under font sizes, use the `rem` values for scalable text and `px` values for non-scalable text. (You can still zoom in with the pixel sized text.) Avoid changing the base styles files (`_mixins.scss`, `_variables.scss` and `theme-llc.scss`). Put all of your customizations and workshop specific styles in workshop.scss. If you'd like to add an improvement or enhancement, please submit a pull request or email [christina@ladieslearningcode.com](mailto:christina@ladieslearningcode.com).
